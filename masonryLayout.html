Considering image display from the beginning

During my work on two printed art publications, I explored possibilities for fair and faithful image layouts in limited space

  • Bluemoon
    Worked collaboratively on a 4-person team to select art for Whitman College's award-winning, annual arts and literature magazine. Worked in Adobe InDesign to create the layout for Edition 34 of the magazine. 
  • PhotoIreland
    Undertook a wide range of responsibilities, including designing the catalog in Adobe InDesign, assisted in constructing exhibition spaces and hanging professional photography, and managed the festival desk while welcoming guests. 

From these explorations I began thinking about how to display images fairly, giving every artist the same amount of real estate on the page. However, I was more interested in creating a systematic way to display images. This was when I started to learn coding and found that the same issues are present on the web.

    Key differences from print
  • fewer limitations on space
  • don’t have to account for gutters, bleed, and trim areas associated with physical printing
  • defining rules to govern image placement rather than manual placement
  • have to decide between existing responsive layout techniques such as css grid, flexbox, or create a custom solution with javascript

Current web solutions don’t do images justice

Current web solutions seem to fall into one of two categories, represented by instagram and unsplash.

  • instagram
    forces images into a square aspect ratio with “object: fill” to make a grid, cropping images in the process
  • unsplash
    uses the more professional solution, splitting the page into columns and forcing images to fit the set width. This typically favors vertically oriented photos and results in photos of radically different sizes

Research for better solutions

As a starting point I looked at the solutions offered by figma, the leading web based design tool for ux (figma reflects the current common web solutions)

    figma
  • Figma’s autolayout is meant to resemble css flexbox as a way to make responsive one dimensional layouts
  • image behavior is based on frames that either fit or fill, cropping images or leaving strange gaps in layouts
  • The Figma community has alternative solutions for vertical and horizontal resizing to maintain aspect ratio, but as impressive as these workarounds are, they are inconsistent and break with new updates

With some knowledge of coding, I new that the solution I imagined was possible, it was just a matter how elaborate the process would be. I researched the best responsive solutions with css and only found more workarounds

    Solution by Kartik Prabhu
  • responsively scales images to fill available space
  • preserves aspect ratio
  • slightly favors horizontal aspect ratios
  • can’t wrap to multiple rows
  • manual input of aspect ratio is required (inelegant)
    Conclusion
  • css alone doesn’t currently support this behavior
  • some degree of javascript necessary
    Desired behavior for my design
  • gives an even amount of real estate to each image
  • responsively scales images to fill available space (no gaps)
  • preserves aspect ratio automatically (no cropping)
  • distributes images to multiple rows
  • css only solution or minimal javascript

Solution

An elegant solution with minimal javascript to work in tandem with flexbox

To compliment the solution by Kartik Prabhu, I wrote some javascript that extends his flexbox design, becoming dynamic, responsive and elegant

    explanation
  • dynamically calculates aspect ratio so hardcoding isn’t necessary
  • javascript converts minHeight to minWidth with aspectRatio
  • minWidth prevents any one image from being excessively shrunken and deprioritized
  • minWidth forces flexbox to wrap to a new row when any image gets too small
    problem -> solution
  • responsive
  • no gaps or unpredictable layouts
  • no cropping images
  • images are given the most fair distribution of space
    limitations
  • vertically oriented images can’t take up full container width if they are the sole image on their row
  • This is only an issue at small screen sizes, where you likely wouldn’t want detailed image galleries anyway

Takeaways

This solution may not be a css one liner, but it uses simple javascript to achieve an impactful result. There are many solutions for column oriented image layouts but there is no reason there shouldn’t be horizontal solutions as well. This project and the research I conducted reveal a potential demand in the community for this type of behavior and this solution could be considered as a new feature added to css in the future.
